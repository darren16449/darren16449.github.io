FinSynGen — LLM-Based Synthetic Financial Time-Series Generator

Python OpenAI Time Series Validation

Problem: Real market data can be insufficient for stress testing and simulation of rare/extreme scenarios.

Approach: Generate synthetic log-return series using LLM prompting, then automatically validate realism with statistical checks.

Architecture: Prompt Engineering → LLM Generation → Statistical Validation → Automated Regeneration → Exportable Dataset

Validation: KS tests, distribution checks, histograms, and quality filtering to reject and regenerate weak samples.

Impact: Enables risk simulation and scenario testing when historical datasets are limited or incomplete.

Demonstrates: Statistical evaluation (DS), pipeline automation (DE), LLM workflow design (ML).

Marketing Content Assistant — LLM + RAG Application

OpenAI RAG ChromaDB Streamlit

Problem: Marketing content needs speed, consistency, and brand control — but generic LLM outputs can drift or hallucinate.

Approach: Use Retrieval-Augmented Generation to ground outputs in relevant context and generate controlled, multi-format content.

Architecture: User Input → Embedding Search (ChromaDB) → Context Injection → GPT Generation → Structured Output

Validation: Prompt templates + format/tone controls to reduce drift, keep outputs consistent, and support multilingual generation.

Impact: Produces grounded, reusable marketing copy with controllable tone, format, and keyword guidance.

Demonstrates: RAG implementation (ML), vector search systems (DE), structured outputs (Analytics).
